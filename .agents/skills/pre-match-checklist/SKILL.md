## 目的

把"赛前容易遗漏什么"的经验显式化为一份**可打印、可勾选、责任到人**的检查清单。避免到了赛场才发现：忘带电池、工具箱锁在实验室、电机备件没带、上电时没人知道该看谁。

不替代人工执行检查，不连接真实硬件，不 Guarante 比赛成绩。**只是把"该带什么、谁负责、按什么顺序做"列清楚，让团队在比赛前一周就能从容准备，而不是出发前一晚慌乱装箱。**

## 触发示例

- "下周要比赛了，生成检查单"
- "给我一份出征清单"
- "要出去打比赛了，看看带什么"
- "生成赛前checklist"
- "区域赛要开始了，准备清单"

## 输入

- **必填**：比赛类型（区域赛/国赛/交流赛/模拟赛）
- **可选**：
  - 出战模块（底盘/机械臂/抬升/视觉等，可勾选）
  - 团队人数
  - 特殊要求（"要去外地，需要额外备件"/"是交流赛，不用带全套"）
- **不需要**：ProbeFlash workspace、远端 API、实时仓库状态

## 协议（执行步骤）

1. **收集信息**。如果用户没提供比赛类型和出战模块，先问这两个关键问题（**不超过 2 个**）。
2. **生成完整检查单**，按模板 8 个章节输出。
3. **空白处用`___`标记**，提示用户填具体责任人/数量。
4. **询问是否需要保存**：
   - **若用户答 yes**：写入 `docs/match-checklists/<date>-<match-type>.md`；目录不存在就创建；写入后回显路径。
   - **若用户答 no 或没回应**：只输出到对话，不写盘。
5. **建议用户打印**：提醒"建议打印装箱checklist（第六节）和 上电仪式（第五节），赛前逐项勾选"

## Prompt 模板要点（用于 AI 实现侧）

生成检查单时强制遵守：

- **模块化装备清单**——按"底盘/机械臂/抬升/视觉"分组，不要混一起。
- **数量留空**——不猜用户有多少电机/电池，用`x<数量>`或`___`标记，让用户填。
- **责任人必须填**——每个检查项、每个分工都留`___`，强制责任到人。
- **便携性考虑**——提醒电池不可托运、贵重物品随身带等航空/交通常识。
- **软件不遗漏**——固件版本确认、配置备份、参数适配等容易忘的软件项。
- **可打印格式**——上电仪式和装箱checklist用表格形式，方便打印后勾选。

## 工具调用

可用 Claude Code 内置工具：

- `Write`：仅在用户同意存档后写入 `docs/match-checklists/<file>.md`

**不调用**：仓库读取、git 操作、远端 API（纯生成类 skill，不需要上下文）。

## 反馈闭环

- 用户反馈"XX没带导致赛场尴尬" → 下次生成时检查单增加该项。
- 用户反馈"XX带了完全没用" → 考虑标记为"可选"或删除。
- 存档文件赛后可人工补充"遗漏/多余"字段，下一届参考。

## 不做的事

- **不替代** 真实装箱动作和责任分配——只是生成模板，执行靠人。
- **不 Guarante** 不遗漏——检查单覆盖率取决于经验积累，不能保证100%。
- **不读取** 用户的实际仓库状态——不检查用户真的有什么电机/代码版本。
- **不写** 用户代码、不发 commit、不 push。
- **不做** 实时库存管理——不追踪"现在仓库还剩几个电机"。

## 与其他 skill 的关系

- 本 skill 是 Bridge facet 的"赛前准备"入口，产出是**一次性**的纸质/电子清单。
- `debug-checklist` 是比赛期间的**急救**入口，产出是排查流程。
- `personal-daily-summary` 是比赛后的**复盘**入口，产出是个人日报/周报。
- 三个 skill 共同覆盖：赛前准备 → 赛中 debug → 赛后复盘 的完整周期。
