# Dogfood 记录

> SKILL-02-DOGFOOD-NOTE 的归宿。备赛期每次 **用了** / **没用** skill 都来这里写 1-3 行；30 天后回看，决定 SKILL.md 的下一轮 prompt 调整方向（SKILL-03）。

## 目的

不是写美化过的"使用心得"。是写**糙的、当下的**反馈，让 skill 真有迭代依据：
- 哪些场景**会自然想到用**？哪些场景**根本没想到**？
- 用过之后，输出**有用**还是**像废话**？
- **没用**的时候是为什么——忘了 / 不信 / 嫌麻烦 / 场景不匹配？

## 文件命名

`YYYY-MM-DD.md`，一天一份；同一天多次记就在同一份里追加。

## 推荐模板（不强制）

```markdown
# 2026-MM-DD

## 用了

- [skill-name] 场景一句话 → 结论一句话（有用 / 没用 / 改了什么 / 出了什么 bug）

## 没用但本可以用

- 场景一句话 → 为什么没想起来 / 为什么不信任 / 为什么嫌麻烦
```

## 边界

- 不写美化过的成功案例。糙比美重要。
- 不写跟 skill 无关的当天日志（那个是 personal-daily-summary skill 的范围）。
- 不打分、不评级、不分析自己的 productivity——这是 SKILL 反馈，不是绩效自评。
