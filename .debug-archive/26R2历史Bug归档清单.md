## 一、CAN通信类问题

### 1. 底盘电机失控（FDCAN FIFO溢出导致HardFault） [优先级：高] [已修复]

**症状**: 底盘有时不受控制，电机像被开环控制，无法闭环调速

**根因分析** (commit 732f31b):
- 当CAN消息接收队列未创建好或队列满时，FIFO中断中直接`return`
- 次数过多会导致FDCAN FIFO持续占满，无法进入电机回调函数
- 电机状态不能更新，控制无法闭环

**修复方案**:
```c
// 在原本直接return的逻辑前添加FIFO排空
if (can_list_queue_handle == NULL) {
    can_list_fdcan_drain_fifo(hfdcan, FDCAN_RX_FIFO0);
    return;
}
```

**验证动作**:
- 检查 `can_list_fdcan_drain_fifo()` 函数是否存在
- 验证队列满时的降级处理逻辑
- 长时间运行测试观察是否还有HardFault

## 三、电机控制类问题

### 5. 达妙双8006协同控制关节震荡 [优先级：高] [已修复]

**症状**: 机械臂用两个8006协同控制的关节总是震荡

**根因分析** (Bug记录.md):
- 速度位置控制模式下，达妙由于零位偏差、CAN指令下发时序、机械结构中心偏移
- 导致一个电机承受绝大部分力（17N vs <1N）
- 当前模式超出扭矩限制后产生震荡

**修复方案**:
- 将达妙切换为PVT（位置-速度-扭矩）控制模式
- 上位机PID参数P值从54压低到10
- P=54时机械运动过程中受力急剧变化会产生过冲校准

**验证动作**:
- 检查双电机电流/扭矩分配是否均衡
- 验证PVT模式配置
- 上位机PID参数确认
